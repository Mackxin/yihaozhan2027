# 壹号栈

一个基于 Vue 3 + Vite 的个人导航网站，集成导航链接、资讯时间线、灵感随记和后台管理。

## 功能

- **首页** — 个人介绍、快捷入口、全局搜索
- **导航** — 分类展示精选网站和工具，支持搜索和分类跳转
- **讯息** — 时间线形式展示资讯，左侧按月份分组，右侧展示详情
- **随记** — Markdown 笔记，支持标签、置顶、搜索、筛选、实时预览、导出，可一键发布到讯息页
- **回忆过往** — 以卡片形式记录生活回忆，支持 Markdown 语法、日期/星期/心情标记，卡片背景色自动轮换。桌面端瀑布流展示，移动端抖音式上下滑动浏览，支持历史记录查看
- **Mac 软件页** — Mac 软件推荐页面，左右布局 + 3 列网格，占屏幕 80% 宽度，支持搜索和后台管理
- **文章管理** — 支持 Markdown 语法写作，自动生成展示页面，可一键发布到导航「万物归一」
- **后台管理** — 可视化管理导航、讯息、Mac软件、文章、回忆数据，支持批量导入、拖拽排序、撤销操作、数据导入导出、自定义导航栏图标、自动备份提醒
- **登录认证** — 后台需要账号密码登录，密码 SHA-256 哈希存储，支持修改密码
- **暗黑模式** — 点击首页标题「壹号棧」切换浅色/深色，持久化记忆
- **响应式** — 适配桌面和移动端，回忆页移动端支持抖音式卡片滑动，Mac 后台移动端自动切换上下布局 + 单列显示

```

├── src/

│   ├── data/           # 默认数据（含 Mac 软件数据）
│   ├── components/     # 组件（登录弹窗、搜索等）
│   └── pages/
│       ├── HomePage.*  # 首页
│       ├── NavPage.*   # 导航页
│       ├── NewsPage.*  # 讯息页
│       ├── NotesPage.* # 随记页
│       ├── MemoryPage.* # 回忆过往页
```

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
# → 浏览器打开 http://localhost:5173

# 构建生产版本
npm run build
```

## 暗黑模式

点击首页标题「壹号栈」即可切换浅色/深色模式，设置自动保存到本地。

## Markdown 实时预览

随记输入框顶部提供「编辑 / 预览」切换按钮，可在写作时实时查看 Markdown 渲染效果，支持标题、加粗、列表、代码块、引用、链接、图片等语法。

## 文章管理

后台「文章」标签支持 Markdown 语法写作：

1. 点击「+ 新建文章」创建文章
2. 输入标题和 Markdown 内容，可切换预览
3. 点击「+ 导航」一键添加到「万物归一」分类
4. 保存后在列表中可以发布/取消发布/编辑/删除

文章数据支持导出/导入和撤销操作。

## 回忆过往

后台「回忆」标签支持以卡片形式记录生活回忆：

1. 点击「+ 新建回忆」创建回忆卡片
2. 选择日期（自动计算星期）、心情 Emoji，输入 Markdown 内容
3. 可切换预览实时查看渲染效果
4. 点击「发布」控制卡片是否在前台可见
5. 前台展示：桌面端瀑布流铺满屏幕，移动端抖音式上下滑动浏览
6. 点击「📋 历史」可查看所有回忆列表，快速跳转

每张卡片自动分配不同的渐变背景色，回忆数据支持导出/导入和撤销操作。

## 数据备份提醒

后台总览页会自动检测上次数据导出时间，根据间隔显示不同级别的提醒横幅：

- **7 天** — 蓝色提示
- **14 天** — 黄色警告
- **30 天** — 红色危险

点击「立即导出」一键备份，或「稍后提醒」关闭横幅。

## 更新网站数据

### 开发模式（推荐）

```bash
# 1. 启动开发服务器
npm run dev

# 2. 在后台点击「生成网站」按钮 → 数据自动写入 public/data.json

# 3. 一键部署（自动构建 + 提交 + 推送）
./deploy.sh
```

### 生产模式

```bash
# 1. 在后台点击「生成网站」按钮，下载 data.json
# 2. 将 data.json 覆盖到 public/data.json
# 3. 一键部署
./deploy.sh
```

> `deploy.sh` 支持自定义提交信息：`./deploy.sh "添加了新链接"`
> 只需构建不提交：`./build.sh`

## 部署（GitHub Pages）

1. 将项目推送到 GitHub 仓库
2. 进入仓库 **Settings → Pages → Source**
3. 选择 `main` 分支的 `/docs` 文件夹
4. 推送代码后 GitHub 自动部署

## 头像设置

将头像图片命名为 `avatar.png`，放到 `public/` 文件夹中，重新构建即可。
