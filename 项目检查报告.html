2仍发现的问题（待你拍板是否修）

后台「导入数据」撤销不完整P1
导入时只把 nav / news 压入撤销栈，但 importAllData 会替换全部模块。撤销后导航/讯息回退、文章/回忆/流记却停在导入后，数据状态互相矛盾。
建议：导入前存一份全量快照，撤销时一次性整体还原。
marked 渲染 v-html 全站未做 XSS 清洗P1
文章 / 回忆 / 流记 / 后台预览均用 v-html="marked(...)" 直接渲染，未 sanitize。当前为「自 XSS」（作者自己录入），但若后台账号泄露或内容未来对外开放，可注入恶意脚本窃取 yihao_admin / localStorage。
建议：引入 DOMPurify，渲染前 DOMPurify.sanitize(marked(content))。
data.json 仅覆盖导航/讯息，与其余模块策略不一致P1
loadRemoteData() 每次刷新用 data.json 覆盖 navCategories 和 timelineItems，但其他模块（流记/回忆/文章…）只存 localStorage。后果：在后台改了导航/讯息，刷新后被 data.json 覆盖丢失；而其余模块不会。行为不一致。
建议：改为「首次以 data.json 为种子写入 localStorage，之后以 localStorage 为准」；或后台编辑能写回 data.json（需后端）。
底部固定 tab 栏遮挡各页底部内容P2
导航页有底部留白补偿，但消息/随记/回忆/工具/文章等滚动容器未预留 56px + 安全区，最后一条记录或按钮被 tab 栏压住、点不到。
建议：各页根滚动容器统一加 padding-bottom: calc(var(--tab-height) + env(safe-area-inset-bottom) + 12px)。
构建产物单包 633KB，未做代码分割P2
所有页面 + 依赖打进一个 chunk，首屏加载偏重（gzip 216KB）。
建议：对各页面 / overlay 用 defineAsyncComponent 懒加载，拆 vendor。
个别字段遍历缺防御（轻微）P2
ToolPage 的 item.desc.toLowerCase()、首页/讯息部分 link.name 等未加 ?.，legacy / 外部注入数据缺字段时可能抛错。
